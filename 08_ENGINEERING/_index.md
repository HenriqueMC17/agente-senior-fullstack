# Diretórios de Engenharia e Arquitetura

Este diretório concentra tudo sobre as práticas, padrões técnicos de software e governança arquitetural de um Agente Senior Full Stack.

## 📂 Pastas e Organização:

- **/ADRs (Architecture Decision Records):**
  Documente suas principais escolhas arquiteturais. Exemplo: A escolha entre usar GraphQL vs REST, banco relacional vs NoSQL. Use o `_Template.md` da pasta.

- **/Best_Practices:**
  Use esta pasta para consolidar estilos e convenções para linguagens específicas (ex: TS, Rust, Python) e metodologias (SOLID, DRY, Clean Code, CQRS).

- **/Code_Snippets:**
  Trechos úteis para reuso e ganho de velocidade (Hooks em React customizados, Utilities de Fetching com SWR/Zustand, Middlewares de autenticação, configs padrão de Caddy/Nginx).

- **/Tech_Radar:**
  Matriz e histórico de tecnologias adotadas (Adopt), em adoção/avaliação (Trial), observação (Assess) ou bloqueio/pausa (Hold). Usar para gerenciar "hype vs pragmatismo".
