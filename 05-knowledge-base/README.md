# 05-knowledge-base

## 🎯 Propósito

Formar o **Cérebro Passivo e Teórico**, ou seja, o santuário da excelência de padrão corporativo. Preserva o material de consulta referencial para embasar e balizar as saídas lógicas de abstração arquitetural tomadas em cenários complexos de infra.

## 🏗️ Papel na Arquitetura (@agente-senior-fullstack)

O Banco de Conhecimentos (Knowledge Base) serve como input RAG (Retrieval-Augmented Generation) dinâmico, elevando consideravelmente o QI do Agente. Oferece guias maduros validados pelo mercado para resolução das mais duras dores de projeto.

## 📂 Natureza dos Arquivos

Guias, Architecture Decision Records (ADRs antigos de projetos transacionados), catálogos técnicos e ensaios técnicos.

- `architecture-patterns.md`: Microfrontends, Microsserviços, CQRS, Event-Sourcing.
- `design-patterns.md`: Padrões do GoF adequados a TS, Python ou Rust.
- `scalability-patterns.md`: Cache Distribuído, CDN Edge, Rate Limiting, Filas e Streams.
- `testing-strategies.md`: Estratégia de Mocking, Fixture, A/B Testing e E2E Playwright.
- `devops-practices.md`: IaC (Infra as Code), Docker hardening, K8s, Observabilidade Zero-Friction.
- `/learning`: Repositório passivo para estudos não implementados via features.
- `/career`: Posicionamento estratégico e frameworks do "Developer to CTO".

## ⚙️ Diretrizes de Manutenção e Evolução

- **Evolução Contínua:** Novas lições aprendidas após post-mortems do projeto e do servidor devem vir para cá (ADRs).
- **Semântica Única:** Este repositório evita divagações e "dicas". Foca-se em resoluções baseadas em dados empíricos de escala.

## 🔗 Relação com Outras Camadas

- Fundamenta os "porquês" contidos nos manifestos concisos do `00-governance`.
- É a resposta principal para quando as limitações e restrições listadas no `06-tech-stacks` entram em conflito sistêmico.
